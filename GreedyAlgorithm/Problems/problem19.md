# Problem 19: Optimal Merge Pattern

## Problem Statement
Given `n` sorted files of different sizes, merge them into a single file. Find the minimum total computation cost to merge all files. The cost of merging two files is the sum of their sizes.

## Input Format
- A list of integers `sizes` representing file sizes.

## Output Format
- An integer representing the minimum total merge cost.

## Constraints
- 2 <= n <= 10^5
- 1 <= sizes[i] <= 10^6

## Example
**Input:** sizes = [2, 3, 4, 5, 6, 7]  
**Output:** 68
